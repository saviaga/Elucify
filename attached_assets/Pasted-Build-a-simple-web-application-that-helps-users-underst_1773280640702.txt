Build a simple web application that helps users understand academic papers by generating structured AI explanations.

The user will upload or paste the text of an arXiv paper. The system analyzes the paper and generates multiple sections that help the reader understand and explore the research.

The interface should look like a research reading dashboard with tabs.

The goal is clarity and usability rather than complex infrastructure.

System behavior

When a user uploads a paper or pastes the text, the system sends the paper to an LLM and generates multiple structured outputs. Each output corresponds to a tab in the interface.

The results should be cached so the sections do not regenerate every time the user switches tabs.

Recommended stack

Frontend
React or Next.js
Tailwind CSS for styling

Backend
Python with FastAPI or Node.js with Express

LLM integration
Use OpenAI API or any compatible LLM API

Paper input

Allow users to either

upload a PDF
paste the full text of the paper
paste an arXiv link

If a PDF or link is provided, extract the text before sending it to the LLM.

Prompt definitions for each section

Use the full paper text as input for each prompt.

Paper

You are given the full text of an academic paper.

Extract and present the core bibliographic information and paper structure.

Return

Title
Authors
Affiliations if available
Publication year
Abstract
Main sections of the paper if identifiable
Keywords or research topics

Do not summarize or reinterpret the paper.

Summary

Explain the academic paper in clear simple language suitable for a 14 year old reader.

In your explanation

Provide a short overview of the paper’s main idea
Describe the research problem
Explain the method in simple terms using analogies if helpful
Summarize the main findings
Explain why the work matters

Avoid technical jargon when possible.

Paper to Video

Write a script for a 2 to 3 minute explainer video about the paper.

Structure

Hook
Problem
Core idea
What the researchers did
What they discovered
Why it matters

Use conversational language.

Whiteboard

Explain the paper as if you are teaching it on a whiteboard.

Break the explanation into steps

The problem being solved
The intuition behind the approach
How the method works step by step
Why this approach improves on previous work

Use simple conceptual explanations and analogies.

Paper Prompts

Generate useful prompts someone could ask an AI assistant to explore this paper further.

Include prompts about

Understanding the method
Comparing the work with previous research
Implementing the approach
Identifying limitations
Extending the research

Return a list of concise questions.

Open Problems

Identify unresolved problems and unanswered questions in the paper.

Focus on

Limitations of the method
Knowledge gaps
Assumptions that might not hold in real world settings
Areas where further research is needed

Write each as a bullet point.

Continue Learning

Identify important concepts someone should study to better understand the paper.

For each concept include

Topic name
Short explanation of why it is relevant

Focus on foundational ideas or prerequisite knowledge.

Related Papers

Identify 5 to 10 related research papers.

Prioritize papers that

Solved a similar problem
Used related methods
Influenced the work

For each paper provide

Title
Short description
How it relates to the original paper

Authors

Analyze the authors of the paper.

For each author include

Name
Primary research areas
Typical topics they publish about

Provide a short overview of the research group.

Collections

Categorize the paper into research topics.

Examples

Machine Learning
Computer Vision
Natural Language Processing
AI Safety
Multimodal Models
Reinforcement Learning
Robotics

Return the most relevant categories and explain briefly why the paper fits each one.

Tweets

Generate 3 to 5 tweet length summaries of the paper.

Each tweet should explain the main idea clearly and highlight what is interesting about the research.

alphaXiv

Rewrite the paper in simplified language similar to a community explained research summary.

Explain key ideas in plain language while preserving technical accuracy.

Clarify motivations, assumptions, and implications.

Glossary

Generate an alphabetical list of advanced domain specific terms that might not be known to an undergraduate computer science student.

For each term provide

the term in bold
a short definition
a verbatim quote from the paper showing how the term is used

Output requirements

Each section should be generated independently and displayed in its corresponding tab.

Ensure the UI is clean and optimized for reading research papers.

Focus on clarity, readability, and structured explanations.

Optional improvements

Allow users to download the generated explanations
Allow sharing a paper page with a link
Allow users to regenerate individual sections if needed.
